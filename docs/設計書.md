## 3. ゲーム仕様（詳細ルール）

> 元ルールは複数人の宴会ゲーム由来。**1人プレイ（対CPU=魔王）** に落とし込む。
> 「写メ」等のリアル要素は **❤️ハート獲得＋演出** に置換する（デジタル化方針）。

### 3-1. サイコロBJ

**目的**: 合計を「目標数」に近づけ、魔王より近い方が勝ち。

1. ラウンド開始時、**目標数（1〜18）をランダム提示**（上部に表示）
2. プレイヤーは **サイコロ1〜3個を選んで振る**を繰り返し、出目を合計に加算
3. **目標数を超えたら即ドボン**（バースト＝負け・BET没収）
4. 「ストップ」で確定
5. 続いて**魔王が同ルールで振る**（CPUは「あと何点必要か」で振る個数を判断）
6. **判定**: 目標数に近い方が勝ち。**ジャスト＝最強（高配当）**。同点はプレイヤー勝ち（任意調整可）
7. **配当**: 勝ち = BET×2、ジャスト勝ち = BET×3、負け/ドボン = BET没収

- BET額は画面下の `BET −/＋` で選択（デフォルト1,000）
- 【将来拡張】原案の「複数人→最も近い人が魔王と対戦」はオンライン対戦として後日

### 3-2. サイコロプロスピ（ダイス野球）

**設定**: 魔王＝ピッチャー、プレイヤー＝バッター。**プレイヤーは打つだけ**（的当て式）。
**目的**: 3アウトになるまでにどれだけ得点できるか（ハイスコア＝コイン獲得）。

**1打席の流れ**:
1. プレイヤーが **左・中・右のうち2つの位置の出目（各1〜6）を予想**
2. さらに **3個の合計値を予想**
3. 魔王が **サイコロ3個を振る**
4. **判定（ヒットの強さ）**:
   - 位置予想が **1つ的中 → ヒット（単打・走者1つ進む）**
   - 位置予想が **2つ的中 → 二塁打（走者2つ進む）**
   - **合計値が的中 → ホームラン（全走者＋打者生還）**
   - どれも外れ → アウト（1アウト加算）
5. **特殊出目（最優先で判定）**:
   - **①①①＝エラー**: 塁に走者なし→打者が一塁へ／走者あり→**全走者1つ進塁＆打者は振り直し**（キャッチャーエラー）
   - **⑥⑥⑥＝三振**: 1アウト
6. 走者が本塁に到達するごとに得点。**3アウトでセット終了**→スコア精算

- 例: 予想`左④ 中③ 計11` → 出目`⑤③⑥` → 中③のみ的中＝**ヒット**、ノーアウト一塁
- ダイヤモンド（塁状況）とアウトカウントを常時表示

### 3-3. スロット（1円スロット555 / 等価スロット777）

**共通の目的**: 指定した位置で数字を3つ揃えて ❤️ハート を獲得。魔王が妨害してくる。

#### ＃555＃版（BJ側・1個振り）
1. 親（魔王）が **サイコロ3個** を振る
2. プレイヤーが **左・中・右のどれか1つの位置を指定**
3. プレイヤーが **サイコロ1個** を振る
4. 指定位置を含め **3つの数字が揃えば → ❤️1個獲得＋当たり演出**
5. 魔王も振り、**揃うのを妨害**してくる

#### ＃777＃版（プロスピ側・2個振り）
1. 親が **サイコロ3個 × 2回** 振り、各位置を**縦に合計**（左=2回の左の和、など）
2. プレイヤーが **左・中・右のどれか1つを指定**
3. プレイヤーが **サイコロ2個** を振る
4. **3つ揃えば → ❤️獲得**（揃え方は2個の和で位置の値に合わせる）
5. 魔王も振って妨害

#### 役テーブル（出目の並びで発動）
| 出目 | 555版の効果 | 777版の効果 |
|------|------------|------------|
| ②②② | 次に自分が振るまで**確変** | — |
| ④④④ | 次に揃えばその枚数ぶん❤️ | — |
| ⑥⑥⑥ | ❤️**+2** | — |
| ⑦⑦⑦ | — | ❤️❤️❤️**確変** |
| ⑩⑩⑩ | — | 次に自分が振るまで**確変** |
| ⑪⑪⑪ | — | 次に揃えばその数ぶん❤️ |
| ⑫⑫⑫ | — | ❤️**+2** |
| ①②③ | **リプレイ**（もう一度振る） | リプレイ |
| ②③④ | 確変前: 振り順が前の人へ戻る | 同左 |
| ③④⑤ | 確変後: 確変が1人ぶん延長 | 同左 |
| ④⑤⑥ | ❤️**+1** | （効果未定・要確認） |
| ⑦⑧⑨ | — | ❤️**+1** |
| ⑧⑨⑩ | — | （効果未定・要確認） |
| ⑩⑪⑫ | — | ❤️**+2** |

- **確変**: 揃いやすくなる/連続ボーナスの状態（実装: 当たり確率↑＋専用演出）。詳細効果は実装時に体感調整
- ⚠️ `④⑤⑥`(777) と `⑧⑨⑩` は原案で効果空欄 → **実装前にユーザー確認**

## 4. 演出設計（静止画ベース＋イベント時に動き＋ボイス）

**方針**: 通常は静止画＋CSS/JSアニメで軽快に。**見せ場のみ**キャラを動かし、事前生成ボイスを鳴らす。

| タイミング | 演出 | 実装 |
|-----------|------|------|
| 通常時（全画面） | 背景静止画＋パーティクル・発光・サイコロ回転 | CSS/JS（軽い） |
| 確変突入 | 魔王が高笑い＋ボイス＋画面フラッシュ（2〜4秒） | 透過PNG切替アニメ＋mp3 |
| ホームラン/ジャスト勝ち | 魔王のリアクション＋効果音＋コイン/❤️飛散 | 同上 |
| ドボン/敗北 | 魔王が勝ち誇る/プレイヤー側の暗転 | 同上 |

- **動画(mp4)は使わない方針が基本**。キャラの透過PNGをCSSで動かす＋ボイスで十分臨場感が出る。どうしても必要な1〜2箇所だけ短い動画を検討
- **ボイス**: 固定セリフを事前にmp3化（10〜20本）。例: 確変「ぐははは、運命は私のものだ！」/ HR被弾「下等な…！」/ 勝利「貴様の負けだ」。再生は無料・使い回し
- すべて**イベント時のみ**・**数秒で通常復帰**・**preloadでカクつき防止**
